# Project Foundation

## Purpose

This directory defines Monad at the project level.

It answers what Monad is, why it exists, what direction it is taking, and what roadmap guides early work.

## Belongs Here

* Product vision.
* Holy Grail vision.
* Product and project charters.
* Strategy.
* Roadmaps.
* Release planning.
* Project-level glossary.

## Does Not Belong Here

* Detailed functional requirements.
* Implementation module details.
* Security standards.
* Work packet issue records.
* Generated context artifacts.
