# WP-E2-015 Handoff

## Status

Complete.

## Product Area

Repository Intelligence, AI-readable context packs, generated context artifacts, and deterministic file export.

## Objective

Add a core export foundation for writing repository context packs to deterministic generated files.

## Completed Outcome

WP-E2-015 extends `crates/monad-core/src/repository_context_pack.rs`.

New constants include:

- `REPOSITORY_CONTEXT_PACK_MARKDOWN_FILENAME`
- `REPOSITORY_CONTEXT_PACK_JSON_FILENAME`

New runtime concepts include:

- `RepositoryContextPackExportedFile`
- `RepositoryContextPackExportResult`
- `default_repository_context_pack_export_dir`
- `export_repository_context_pack`

A new root helper is also available:

- `export_repository_context_pack_from_workspace`

Default export directory:

- `.monad/context/generated/`

Generated files:

- `.monad/context/generated/repository-context-pack.md`
- `.monad/context/generated/repository-context-pack.json`

## Important Implementation Notes

This slice is core-only.

This slice does not add a CLI `--write` flag.

The export foundation writes both Markdown and JSON in one deterministic operation.

The generated context directory is intentionally reviewable and repo-local.

The next slice should expose this through the CLI.

## Expected Result After Verification

Expected verification commands:

- `cargo fmt`
- `cargo test`
- `tools/scripts/verify.sh`

Expected successful result:

- context pack export tests pass;
- generated Markdown and JSON files are written in test fixtures;
- verification finishes with `Verification baseline passed.`.
