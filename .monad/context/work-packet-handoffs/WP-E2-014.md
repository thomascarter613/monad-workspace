# WP-E2-014 Handoff

## Status

Complete.

## Product Area

Repository Intelligence, AI-readable context packs, CLI command surface, and verification.

## Objective

Expose repository context-pack rendering through a thin `monad context` CLI command.

## Completed Outcome

WP-E2-014 adds the `monad context` command.

Supported context command forms:

- `monad context`
- `monad context --format=markdown`
- `monad context --format=json`
- `monad context --format=md`
- `monad context --format=text`

The CLI remains thin.

Context-pack construction and rendering remain in `monad-core`.

`tools/scripts/verify.sh` now includes smoke tests for context Markdown, JSON, `md`, and `text` output paths.

## Important Implementation Notes

Normal commands still support only text and JSON.

The graph command has graph-specific render formats.

The context command has context-specific render formats.

The context command defaults to Markdown because the repository context pack is designed for human and LLM reading.

This slice does not write context packs to files yet.

This slice does not add context-pack export destinations, redaction, summarization budgets, or model-provider integration.

## Expected Result After Verification

Expected verification commands:

- `cargo fmt`
- `cargo test`
- `cargo run --quiet -p monad-cli -- context`
- `cargo run --quiet -p monad-cli -- context --format=json`
- `tools/scripts/verify.sh`

Expected successful result:

- Markdown context output begins with `# Monad Repository Context Pack`;
- JSON context output includes `repository_context_pack`;
- verification finishes with `Verification baseline passed.`.

## Continuation Recommendation

Proceed next to WP-E2-015.

Recommended next slice:

WP-E2-015 — Add Repository Context Pack Export Foundation

Suggested direction:

- add a core file export helper for context packs;
- support writing Markdown and JSON context packs to `.monad/context/generated/`;
- keep paths deterministic and reviewable;
- add `monad context --write` only if the core export model is stable enough;
- otherwise keep WP-E2-015 core-only and expose CLI writing in WP-E2-016.
