# WP-E2-005 Handoff

## Status

Complete.

## Product Area

Repository Intelligence, traversal planning, traversal safety, and context continuity.

## Objective

Add recursive traversal planning types and conservative guardrails before implementing any deep repository traversal.

## Completed Outcome

WP-E2-005 adds a future traversal planning model in `monad-core`.

New runtime concepts include:

- `RepositoryTraversalMode`
- `RepositoryTraversalDecision`
- `RepositoryTraversalGuardrails`
- `RepositoryTraversalPlanEntry`
- `RepositoryTraversalPlan`
- `build_traversal_plan`

The traversal guardrails are conservative by default:

- max depth is `3`;
- symlink traversal is disabled;
- generated/external entries are excluded;
- ignore files are expected to be respected;
- deterministic ordering is required.

`monad inspect` now includes future traversal guardrails and per-entry future traversal decisions through the existing output layer.

## Important Implementation Notes

This work packet does not implement recursive traversal.

The inspection model remains shallow and deterministic.

The CLI remains thin and unchanged.

The next slice may implement bounded traversal using the plan and guardrails created here.

## Expected Result After Verification

Expected verification commands:

- `cargo fmt`
- `cargo test`
- `cargo run --quiet -p monad-cli -- inspect`
- `cargo run --quiet -p monad-cli -- inspect --format=json`
- `tools/scripts/verify.sh`

Expected successful result:

- text output includes `future_traversal_guardrails:`;
- JSON output includes `future_traversal`;
- generated/external paths are planned as `skip_by_default`;
- safe roots are planned as `candidate_for_future_traversal`;
- verification finishes with `Verification baseline passed.`.

WP-E2-006 — Implement Bounded Repository Traversal Foundation
