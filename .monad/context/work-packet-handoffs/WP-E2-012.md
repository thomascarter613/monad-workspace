# WP-E2-012 Handoff

## Status

Complete.

## Product Area

Repository Intelligence, advisory policy checks, dependency signals, bounded traversal, and inspect output.

## Objective

Add advisory repository-intelligence policy diagnostics to `monad-core`.

## Completed Outcome

WP-E2-012 adds `crates/monad-core/src/repository_policy.rs`.

New runtime concepts include:

- `RepositoryPolicySeverity`
- `RepositoryPolicyDiagnostic`
- `RepositoryPolicyReport`
- `evaluate_repository_intelligence_policy`

Initial policy diagnostics include:

- missing top-level README signal;
- missing top-level license signal;
- dependency manifest signal without lockfile signal;
- generated/external traversal safety information;
- bounded traversal max-depth information.

`monad inspect` now includes policy metrics and policy diagnostics through the existing output layer.

## Important Implementation Notes

Policy checks are advisory-only in this slice.

This slice does not make `monad check` fail on repository-intelligence policy diagnostics.

This slice does not invoke external tools.

This slice does not parse dependency contents.

The next slice can promote policy diagnostics into `monad check` or build AI-readable repository context packs.

## Expected Result After Verification

Expected verification commands:

- `cargo fmt`
- `cargo test`
- `cargo run --quiet -p monad-cli -- inspect`
- `cargo run --quiet -p monad-cli -- inspect --format=json`
- `tools/scripts/verify.sh`

Expected successful result:

- text output includes `policy:`;
- JSON output includes `policy`;
- repository policy tests pass;
- verification finishes with `Verification baseline passed.`.
