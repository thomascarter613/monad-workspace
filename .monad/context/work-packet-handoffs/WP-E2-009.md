# WP-E2-009 Handoff

## Status

Complete.

## Product Area

Repository Intelligence, graph rendering, CLI command surface, and verification.

## Objective

Expose repository graph rendering through a thin `monad graph` CLI command.

## Completed Outcome

WP-E2-009 adds the `monad graph` command.

Supported graph command forms:

- `monad graph`
- `monad graph --format=json`
- `monad graph --format=mermaid`
- `monad graph --format=dot`

The CLI remains thin.

Graph construction and rendering remain in `monad-core`.

`tools/scripts/verify.sh` now includes smoke tests for graph text, JSON, Mermaid, and DOT output.

## Important Implementation Notes

Normal commands still support only text and JSON.

The graph command has its own graph render format parser because graph output supports Mermaid and DOT in addition to text and JSON.

This slice does not introduce graph depth flags, traversal flags, or graph semantic enrichment.

## Expected Result After Verification

Expected verification commands:

- `cargo fmt`
- `cargo test`
- `cargo run --quiet -p monad-cli -- graph`
- `cargo run --quiet -p monad-cli -- graph --format=json`
- `cargo run --quiet -p monad-cli -- graph --format=mermaid`
- `cargo run --quiet -p monad-cli -- graph --format=dot`
- `tools/scripts/verify.sh`

Expected successful result:

- text graph output begins with `Monad repository graph`;
- JSON graph output includes `repository_graph`;
- Mermaid graph output starts with `flowchart TD`;
- DOT graph output starts with `digraph repository`;
- verification finishes with `Verification baseline passed.`.
