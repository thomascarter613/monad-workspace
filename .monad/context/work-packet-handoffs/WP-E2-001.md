# WP-E2-001 Handoff

## Work Packet

WP-E2-001 — Establish Repository Inspection Foundation

## Current Epic

E2 — Repository Intelligence Foundation

## Prior Work

E0 — Project Foundation is complete.

E1 — Runtime Foundation is complete.

## Objective

Add the first repository inspection model and runtime behavior so Monad can summarize top-level workspace structure.

## Starting Point

The runtime already has:

* `WorkspaceContext`;
* `MonadManifest` loading;
* `DiagnosticReport`;
* `MonadError`;
* `run_workspace_checks`;
* `RepositoryContract`;
* `OutputFormat`;
* text and JSON output;
* `monad info`;
* `monad check`.

## Recommended Implementation Direction

Create a new `monad-core` module for repository inspection.

The first implementation should inspect top-level workspace entries, distinguish files from directories, avoid deep traversal, and prepare typed output for future JSON rendering and graph work.

## Rules

* Use Rust Apprenticeship Mode.
* Provide complete file contents.
* Add beginner-readable comments.
* Keep CLI thin.
* Put durable runtime logic in `monad-core`.
* Add tests.
* Run verification.
* Use `python3`, not `python`.
* Do not introduce Bazel, Pants, Buck2, or Nx.

## Expected Verification

```bash
cargo fmt --check
cargo test
tools/scripts/verify.sh
```
