# WP-E2-017 Handoff

## Status

Complete.

## Product Area

Repository Intelligence, generated context artifacts, ignore policy, advisory policy diagnostics, and verification.

## Objective

Encode the policy decision that `.monad/context/generated/` is generated output and should not be committed by default.

## Completed Outcome

WP-E2-017 adds the following `.gitignore` rule:

- `.monad/context/generated/`

WP-E2-017 extends `crates/monad-core/src/repository_policy.rs`.

New constants include:

- `GENERATED_CONTEXT_ARTIFACT_DIR`
- `GENERATED_CONTEXT_ARTIFACT_IGNORE_RULE`

New policy diagnostic codes include:

- `MONAD-RI-0300` — generated context artifact directory exists;
- `MONAD-RI-0301` — generated context artifacts should be ignored by default;
- `MONAD-RI-0302` — generated context artifacts are ignored by default.

`tools/scripts/verify.sh` now checks that `.monad/context/generated/` is ignored.

## Important Implementation Notes

Generated context artifacts are deterministic and reviewable, but ignored by default.

This does not prevent a user from intentionally force-adding generated artifacts.

This policy can later become configurable if Monad adds a generated artifact publication workflow.

The verifier still removes `.monad/context/generated/` after the context write smoke test.

Expected verification commands:

Expected successful result:

- the ignore rule exists;
- the verifier checks the ignore rule;
- verification finishes with `Verification baseline passed.`.
